  <name>Task 1: main.py — app wiring with lifespan, router, StaticFiles</name>
  <files>main.py</files>
  <action>
Create main.py. Follow this exact structure (dictated by FastAPI's StaticFiles catch-all behavior):

```python
from contextlib import asynccontextmanager
from fastapi import FastAPI
from fastapi.staticfiles import StaticFiles
from routers import config as config_router
from config import load_settings

@asynccontextmanager
async def lifespan(app: FastAPI):
    settings = load_settings()
    settings.cache_dir.mkdir(parents=True, exist_ok=True)
    settings.output_dir.mkdir(parents=True, exist_ok=True)
    yield

app = FastAPI(title="Drill Clip Extractor", lifespan=lifespan)

# CRITICAL: Include routers BEFORE StaticFiles mount.
# StaticFiles is a catch-all — any router included after the mount
# will be intercepted and return index.html instead of JSON.
app.include_router(config_router.router, prefix="/api")

# StaticFiles MUST be last — it is the catch-all fallback.
app.mount("/", StaticFiles(directory="static", html=True), name="static")
```

Do NOT use `@app.on_event("startup")` — it is deprecated in FastAPI 0.93+ and raises warnings on 0.135. Use the lifespan context manager shown above.

Verify the router order is enforced by an integration test in tests/test_main.py:
- GET /api/config returns application/json (not text/html)
- GET /api/health returns {"status": "ok"}
- GET / returns text/html (index.html served by StaticFiles)

Create static/index.html. Requirements:
- Single page, no frameworks, no build step, no external CDN imports
- Represents the eventual UI structure for Phase 3 (gives the user something real to look at, not just a placeholder)
- Must include the semantic structure Phase 3 will populate: URL input section, clip form section (start/end timestamps, filename), session list section
- All form fields must be keyboard-navigable (proper tab order, no tabindex=-1 on interactive elements)
- Minimal inline CSS only — no stylesheet link, no external resources

Structure:
```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Drill Clip Extractor</title>
  <style>
    /* minimal: body font, max-width container, section spacing */
    /* no animations, no complex components */
  </style>
</head>
<body>
  <main>
    <h1>Drill Clip Extractor</h1>

    <!-- Section 1: Source URL -->
    <section id="source-section">
      <label for="url-input">Instructional URL</label>
      <input type="url" id="url-input" placeholder="https://youtube.com/watch?v=...">
      <button type="button" id="load-btn">Load</button>
    </section>

    <!-- Section 2: Clip form -->
    <section id="clip-section">
      <form id="clip-form">
        <label for="start">Start</label>
        <input type="text" id="start" placeholder="0:00" pattern="\d+:\d{2}(:\d{2})?">
        <label for="end">End</label>
        <input type="text" id="end" placeholder="0:30" pattern="\d+:\d{2}(:\d{2})?">
        <label for="filename">Filename</label>
        <input type="text" id="filename" placeholder="armbar-setup">
        <button type="submit">Extract Clip</button>
      </form>
    </section>

    <!-- Section 3: Session clip list (populated by JS in Phase 3) -->
    <section id="session-section">
      <h2>Session Clips</h2>
      <ul id="clip-list">
        <!-- clips rendered here by JS in Phase 3 -->
      </ul>
    </section>

    <!-- Status / progress area (used by SSE in Phase 3) -->
    <div id="status" aria-live="polite"></div>
  </main>

  <script>
    // Phase 1: Fetch and display current config on load
    // Wires up the config API so the shell is live, not purely static
    async function loadConfig() {
      try {
        const res = await fetch('/api/config');
        const cfg = await res.json();
        document.getElementById('status').textContent =
          'Output: ' + cfg.output_dir;
      } catch (e) {
        document.getElementById('status').textContent = 'Config load failed: ' + e.message;
      }
    }
    document.addEventListener('DOMContentLoaded', loadConfig);
  </script>
</body>
</html>
```

Key UX constraints:
- No `tabindex` manipulation — natural DOM order provides correct keyboard flow
- `<form id="clip-form">` with `<button type="submit">` so Enter submits (Phase 3 wires the submit handler)
- `aria-live="polite"` on status div so screen readers announce progress updates
- No external resources — tool must work without internet access after initial download
